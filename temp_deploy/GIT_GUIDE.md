# 📚 دليل Git - نظام التحكم في الإصدارات

## 🎯 ما هو Git؟

Git هو نظام تحكم في الإصدارات الموزع (Distributed Version Control System) يساعد المطورين على تتبع التغييرات في الكود وإدارة المشاريع بكفاءة.

---

## 🚀 فوائد استخدام Git

### 1. **تتبع التغييرات 📝**
- **تسجيل كل تغيير**: يحفظ Git كل تعديل تم على الملفات مع التاريخ والمؤلف
- **العودة لأي نقطة**: يمكنك العودة لأي إصدار سابق من المشروع
- **مقارنة الإصدارات**: رؤية الفرق بين أي إصدارين

### 2. **العمل الجماعي 👥**
- **التعاون الآمن**: عدة مطورين يمكنهم العمل على نفس المشروع
- **دمج التغييرات**: دمج التعديلات من فرق مختلفة بأمان
- **حل التعارضات**: أدوات ذكية لحل التعارضات عند دمج الكود

### 3. **النسخ الاحتياطي ☁️**
- **نسخ محلي و سحابي**: نسخة محلية + نسخة على GitHub/GitLab
- **الحماية من الفقدان**: حتى لو فُقد الحاسوب، الكود آمن
- **النسخ المتعدد**: يمكن حفظ المشروع في عدة أماكن

### 4. **التطوير الآمن 🛡️**
- **التجربة الآمنة**: يمكن تجربة ميزات جديدة في فروع منفصلة
- **العزلة**: التغييرات في فرع لا تؤثر على الفرع الرئيسي
- **الدمج الذكي**: دمج الميزات المكتملة فقط

### 5. **إدارة المشاريع 📊**
- **تتبع المهام**: ربط التغييرات بالمهام والمشاكل
- **إحصائيات المساهمة**: معرفة من ساهم في كل جزء
- **الإصدارات**: إنشاء إصدارات رسمية من التطبيق

## 🛠️ الأوامر الأساسية في Git

### إعداد المشروع
```bash
git init                    # إنشاء repository جديد
git clone <url>            # نسخ repository موجود
```

### إدارة التغييرات
```bash
git add .                  # إضافة جميع الملفات للـ staging
git add <file>            # إضافة ملف محدد
git commit -m "رسالة"     # حفظ التغييرات مع رسالة
```

### الفروع (Branches)
```bash
git branch                 # عرض الفروع
git branch <name>         # إنشاء فرع جديد
git checkout <name>       # الانتقال لفرع
git merge <branch>        # دمج فرع مع الفرع الحالي
```

### العمل مع السحابة
```bash
git remote add origin <url>    # ربط مع repository سحابي
git push origin master         # رفع التغييرات للسحابة
git pull origin master         # تحميل التغييرات من السحابة
```

## 📋 مثال عملي: سير عمل التطوير

### السيناريو: إضافة ميزة جديدة

```bash
# 1. إنشاء فرع للميزة الجديدة
git checkout -b feature/new-pdf-compression

# 2. العمل على الميزة
# ... تعديل الملفات ...

# 3. حفظ التغييرات
git add .
git commit -m "Add local PDF compression fallback"

# 4. رفع الفرع للسحابة
git push origin feature/new-pdf-compression

# 5. إنشاء Pull Request لدمج الميزة
# (عبر GitHub/GitLab interface)

# 6. بعد المراجعة، دمج الميزة
git checkout master
git merge feature/new-pdf-compression
```

## 📈 خطوات التطوير المستقبلية

### 1. **ربط مع GitHub**
```bash
# إنشاء repository على GitHub
# ثم ربطه محلياً:
git remote add origin https://github.com/username/an-academy.git
git push -u origin master
```

### 2. **إنشاء فروع للميزات**
```bash
git checkout -b feature/ai-improvements
git checkout -b feature/mobile-responsive
git checkout -b bugfix/pdf-compression
```

### 3. **إدارة الإصدارات**
```bash
git tag v1.0.0  # إصدار رسمي
git tag v1.1.0  # إصدار مع التحسينات
```
